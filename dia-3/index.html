🎬 Mi Catálogo de Películas

Construye tu propio sistema tipo "mini Netflix": un programa con menú do/while para gestionar tu catálogo de pelis. Lo armas por etapas en un archivo catalogo.js, aplicando TODO lo visto: funciones arrow, arreglo de objetos, find, filter, reduce, cambio de estado y validaciones. Versionas con Git (un commit por etapa), subes a GitHub y publicas en GitHub Pages. Esto NO es ningún sistema que hicimos en clase: te toca aplicar las herramientas a algo nuevo.

📜 Cómo se juega

  1. Primero arma tu proyecto (sección de abajo): carpeta, catalogo.js, index.html y repositorio Git.
  2. Resuelve las etapas en orden: cada una agrega una función al mismo catalogo.js.
  3. La misión dice QUÉ lograr, no CÓMO. Si te atascas, abre la 💡 pista.
  4. Prueba cada función con console.log en tu computador (Node o Live Server + consola F12) y compara con la salida esperada.
  5. Cuando una etapa funcione: haz un commit. Una etapa = un commit.
  6. Marca "Etapa cumplida" y mira subir tu progreso.

Etapa 1 · El catálogo inicial Nivel 1

Empieza por los datos. Copia el arreglo de películas y aprende a recorrerlo.

  1. Copia el arreglo peliculas al inicio de tu catalogo.js. Cada película es un objeto con título, género, duración (en minutos) y vista (si ya la viste o no).
  2. Recórrelo con forEach e imprime una línea por película con su número, título, género y duración.
¿Necesitas una pista?
  • peliculas.forEach((p, i) => console.log(...)) — el segundo parámetro i es el índice.
  • Recuerda mostrar i + 1 para numerar desde 1.
  • Arma el texto con un template literal o con +: p.titulo, p.genero, p.duracion.
catalogo.js — punto de partida
const peliculas = [
  { titulo: "Matrix",    genero: "Acción",    duracion: 136, vista: false },
  { titulo: "Coco",      genero: "Animación", duracion: 105, vista: false },
  { titulo: "Inception", genero: "Acción",    duracion: 148, vista: false },
];

// Recorre 'peliculas' con forEach e imprime nº, título, género y duración.

1. Matrix (Acción, 136 min)
2. Coco (Animación, 105 min)
3. Inception (Acción, 148 min)

El formato exacto lo eliges tú; deben aparecer las 3 películas con sus datos.

Al terminar: git add catalogo.js → git commit -m "feat: etapa 1 - catalogo inicial"

Etapa 3 · Marcar como vista (find + cambiar estado) Nivel 3

Buscar una película por su título y cambiar su estado a vista. Con validación si no existe.

  1. Crea marcarVista(titulo) que busque la película por su título y ponga su vista en true.
  2. Si NO existe una película con ese título, avisa con un mensaje y no hagas nada más (guard clause).
  3. Prueba marcando "Matrix" (existe) y "Avatar" (no existe).
¿Necesitas una pista?
  • Para buscar UNO: peliculas.find((p) => p.titulo === titulo).
  • find devuelve la película encontrada o undefined si no hay. Valida con if (!peli) { ...; return; }.
  • Cambiar el estado: peli.vista = true (modificas la propiedad del objeto encontrado).

Etapa 6 · Tiempo total visto (filter + reduce) Nivel 3

¿Cuántos minutos llevas viendo pelis? Combina filter (solo las vistas) con reduce (sumar duraciones).

  1. Crea tiempoTotalVisto() que sume la duracion de solo las películas vistas.
  2. Marca "Matrix" e "Inception" como vistas y muestra el total de minutos.
¿Necesitas una pista?

  • Puedes encadenar filter y reduce en una sola línea.
  • Matrix (136) + Inception (148) = el total que debes obtener.

🏆 Jefe Final · El menú do/while Integrador

Hasta ahora tienes funciones sueltas. Ahora construye el menú interactivo que las usa todas, como un programa real.

Arma un menú con do/while que muestre las opciones y, según lo que elija el usuario, llame a tus funciones. El menú se repite hasta que elija Salir. Opciones mínimas:

  1. Agregar película (pide título, género y duración).
  2. Ver todas (con una marca [✓] / [ ] según si está vista).
  3. Marcar como vista (pide el título).
  4. Filtrar por género (pide el género).
  5. Recomendar (muestra las no vistas).
  6. Tiempo total visto.
  7. Salir.

Requisito: el menú NO repite lógica; solo invoca las funciones que ya creaste. Para leer la opción usa prompt() (o la función pedir() simulada que viste en clase para probar sin ventanitas).

¿Necesitas una pista?
  • Estructura: let opcion; do { ...muestra menú; opcion = prompt("Opción:"); if/else if... } while (opcion !== "7");.
  • Usa do/while (no while) para que el menú aparezca al menos una vez.
  • Para probar sin ventanitas, copia el patrón pedir() con entradas simuladas que usamos en clase y, en tu PC real, cámbialo por prompt.
  • Cada opción del if/else if llama a UNA de tus funciones. Nada de reescribir cálculos.
