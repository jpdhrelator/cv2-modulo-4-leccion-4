🏆 Mini Software: Ranking de Torneo Gamer

Vas a construir una app de verdad en JavaScript: un sistema que procesa los puntajes de un torneo de videojuegos. La construyes por etapas en un solo archivo torneo.js, usando las arrow functions de muchas formas: como funciones utilitarias, para devolver objetos, como callback de tus propias funciones y con valor por defecto. Más return, guard clauses y objetos. Versionas con Git (un commit por etapa), subes a GitHub y publicas en GitHub Pages.

📜 Cómo se juega

  1. Primero arma tu proyecto (sección de abajo): carpeta, torneo.js, index.html y repositorio Git.
  2. Resuelve las etapas en orden: cada una agrega una función al mismo torneo.js y usa las anteriores.
  3. La misión dice QUÉ lograr, no CÓMO. Si te atascas, abre la 💡 pista.
  4. Pruébalo en tu computador (Node o Live Server + consola F12) y compara con la salida esperada.
  5. Cuando una etapa funcione: haz un commit. Una etapa = un commit con mensaje claro.
  6. Marca "Etapa cumplida" y mira subir tu progreso.

Abre la terminal dentro de torneo-gamer/:

git init
git add .
git commit -m "chore: estructura inicial del proyecto"
vas a construir la app paso a paso y harás un commit por cada etapa que funcione. Tu historial debe contar la historia: "agregué total", "agregué ranking", "agregué medallas"... NO un solo commit gigante al final.

4 Construye y commitea etapa por etapa

Agregas la función de la etapa a torneo.js, la pruebas y, cuando funciona, la versionas:

Revisa tu avance con git status y git log --oneline.

Después de cada nuevo commit, súbelo con git push.

Parte 1 · Cimientos

Datos y funciones utilitarias (arrow)

Etapa 1 · Los jugadores del torneo Nivel 1

Todo software empieza por sus datos. Aquí tienes el dataset: cópialo tal cual al inicio de tu torneo.js.

  1. Copia el arreglo jugadores a tu archivo. Cada jugador es un objeto con nombre, país y un arreglo de puntajes (3 partidas).
  2. Recórrelo e imprime una línea por jugador con su nombre y su país.
¿Necesitas una pista?
  • Para acceder a los datos de cada jugador: jugador.nombre, jugador.pais.
  • Puedes recorrer con un for clásico o con forEach. Usa un template literal `...${j.nombre}...`.
torneo.js — punto de partida
const jugadores = [
  { nombre: "Aragorn", pais: "CL", puntajes: [120, 90, 150] },
  { nombre: "Legolas", pais: "AR", puntajes: [200, 180, 220] },
  { nombre: "Gimli",   pais: "CL", puntajes: [80, 60, 100] },
  { nombre: "Gandalf", pais: "PE", puntajes: [300, 250, 290] },
];

// Recorre 'jugadores' e imprime el nombre y el país de cada uno.

Aragorn (CL)
Legolas (AR)
Gimli (CL)
Gandalf (PE)

El formato del texto lo eliges tú; deben aparecer los 4 jugadores con su país.

Al terminar: git add torneo.js → git commit -m "feat: etapa 1 - datos de jugadores"

Etapa 2 · Funciones utilitarias con arrow Nivel 2

Tus primeras arrow functions: dos herramientas de cálculo cortitas, una apoyándose en la otra.

  1. Crea una arrow function puntajeTotal(jugador) que devuelva la suma de sus puntajes. (Aquí sí conviene reduce, que ya conoces.)
  2. Crea otra arrow promedio(jugador) que devuelva el promedio, reutilizando puntajeTotal (no vuelvas a sumar).
  3. Muestra el total y el promedio de dos jugadores.
¿Necesitas una pista?
  • Arrow con return implícito: const promedio = (j) => puntajeTotal(j) / j.puntajes.length;.
  • Para sumar, reduce((acc, p) => acc + p, 0) sobre jugador.puntajes.
  • Fíjate cómo una arrow (promedio) usa a otra (puntajeTotal): funciones que colaboran.
torneo.js — agrega debajo
// const puntajeTotal = (jugador) => ...
// const promedio = (jugador) => ...   (reutiliza puntajeTotal)

Gandalf: total 840, promedio 280

Etapa 3 · Una arrow que devuelve un objeto Nivel 3

Otra forma de usar arrow: que su resultado sea un objeto. Cuidado, que aquí hay una trampa famosa.

  1. Crea crearResumen(jugador) que devuelva un objeto con tres propiedades: nombre, total y promedio (usando tus funciones de la etapa 2).
  2. Hazlo con return implícito (sin escribir return).
  3. Muestra el resumen de Gandalf.
¿Necesitas una pista?
  • La trampa: (j) => { nombre: j.nombre } NO funciona (JS cree que {} es el cuerpo).
  • Envuelve el objeto en paréntesis para devolverlo: (j) => ({ nombre: j.nombre, ... }).
  • Las propiedades total y promedio salen de llamar a tus funciones anteriores.
torneo.js — agrega debajo
// const crearResumen = (jugador) => ({ ... });   // ¡ojo con los () !


// Muestra el resumen de Gandalf

{ nombre: 'Gandalf', total: 840, promedio: 280 }

Si te sale undefined, te faltaron los paréntesis envolviendo el objeto. Es el error N°1 con arrow.

Al terminar: git add torneo.js → git commit -m "feat: etapa 3 - arrow que retorna objeto"

Parte 2 · Tus funciones reciben funciones

Arrow como callback de TUS propias funciones

Etapa 4 · Tu propio "para cada jugador" Nivel 3

¿Recuerdas el ejecutador de la clase, que recibía una función y la ejecutaba? Vas a construir el tuyo. Una función que recibe otra función (una arrow) como argumento.

  1. Crea paraCadaJugador(accion) que reciba una función accion y la ejecute con cada jugador de la lista (recorre con un for por dentro).
  2. Pruébala pasándole una arrow distinta cada vez: una que imprima solo el nombre, y otra que imprima nombre + país.

Esto es lo mismo que hace forEach por dentro: ahora lo entiendes porque lo construiste tú.

¿Necesitas una pista?
  • accion es una función: dentro del for la ejecutas con accion(jugadores[i]).
  • Al usarla, le pasas la arrow en el momento: paraCadaJugador((j) => console.log(j.nombre)).
  • NO pongas paréntesis a la arrow cuando la pasas: pasas la función, no su resultado.
torneo.js — agrega debajo
// const paraCadaJugador = (accion) => {
//   recorre 'jugadores' con un for y ejecuta accion(jugadores[i])
// };


// Pruébala con DOS arrow distintas:
// 1) que imprima solo el nombre
// 2) que imprima nombre + país

-- solo nombre --
Aragorn
Legolas
Gimli
Gandalf
-- nombre y país --
Aragorn - CL
Legolas - AR
Gimli - CL
Gandalf - PE

Etapa 5 · Arrow como "condición" (predicado) Nivel 3

Ahora una arrow que devuelve true o false, y se la pasas a tu propia función para que decida. A eso se le llama predicado.

  1. Crea contarJugadoresQue(condicion) que reciba una arrow condicion (que devuelve true/false) y devuelva cuántos jugadores la cumplen (cuenta con un for por dentro, sin usar filter).
  2. Pruébala con dos arrow-condición distintas: una que sea "su total es 400 o más" y otra que sea "su país es CL".
¿Necesitas una pista?
  • Dentro del for: if (condicion(jugadores[i])) cuenta++;.
  • Las condiciones se pasan como arrow: contarJugadoresQue((j) => puntajeTotal(j) >= 400).
  • La otra: (j) => j.pais === "CL". ¡Una arrow que devuelve un booleano!
torneo.js — agrega debajo
// const contarJugadoresQue = (condicion) => {
//   recorre con for, suma 1 cuando condicion(jugador) sea true, y retorna la cuenta
// };


// Prueba 1: cuántos tienen total >= 400
// Prueba 2: cuántos son de país "CL"

Con 400+ puntos: 2
Del país CL: 2

Total >=400: Legolas y Gandalf. País CL: Aragorn y Gimli. La MISMA función responde preguntas distintas según la arrow que le pases.

Parte 3 · Decisiones

Valor por defecto y guard clauses

Etapa 6 · Clasificar con valor por defecto Nivel 2

Una arrow con cuerpo de varias líneas, un parámetro con valor por defecto y return temprano.

  1. Crea clasificar(jugador, minimo) que devuelva "Clasificado" si el total del jugador alcanza el mínimo, o "Eliminado" si no.
  2. El parámetro minimo debe valer 400 por defecto.
  3. Prueba: Aragorn sin mínimo (usa 400), Gandalf sin mínimo, y Aragorn pasando 300.
¿Necesitas una pista?
  • Arrow con llaves y valor por defecto: const clasificar = (jugador, minimo = 400) => { ... }.
  • Con llaves, el return es obligatorio: if (puntajeTotal(jugador) >= minimo) return "Clasificado";.
  • El valor por defecto entra solo cuando NO pasas el segundo argumento.
torneo.js — agrega debajo
// const clasificar = (jugador, minimo = 400) => { ... return ... };


// Prueba: Aragorn (sin mínimo), Gandalf (sin mínimo), Aragorn con 300

Aragorn: Eliminado
Gandalf: Clasificado
Aragorn (mínimo 300): Clasificado

Aragorn tiene 360: con el mínimo por defecto (400) queda Eliminado, pero bajando el mínimo a 300, Clasifica.

Al terminar: git add torneo.js → git commit -m "feat: etapa 6 - clasificar con valor por defecto"

Etapa 7 · Medalla por puntaje (guard clauses) Nivel 2

Una arrow que toma una decisión con varios return en cadena, según el puntaje.

  1. Crea medalla(total) que devuelva: "🥇 Oro" si el total es 800 o más; "🥈 Plata" si es 500 o más; "🥉 Bronce" si es 300 o más; y "Sin medalla" en otro caso.
  2. Usa return temprano (sin else anidados).
  3. Usando tu paraCadaJugador de la etapa 4, imprime la medalla de cada jugador.
¿Necesitas una pista?
  • Encadena if con return: if (total >= 800) return "🥇 Oro"; y sigue bajando.
  • El orden importa: revisa de mayor a menor umbral.
  • Reutiliza la etapa 4: paraCadaJugador((j) => console.log(... medalla(puntajeTotal(j)) ...)).

Jefe Final

El reporte que junta TODO

🏆 Jefe Final · El reporte del torneo Integrador

La función que coordina a todas las demás. Aquí demuestras que tus funciones colaboran como un equipo: una sola llamada genera el informe completo.

Crea una arrow reporteTorneo que, usando las funciones que ya creaste, imprima un informe completo con TODO esto:

  1. Un título del torneo.
  2. Una línea por jugador (recórrelos con tu paraCadaJugador de la etapa 4) que muestre: nombre, total, promedio, su medalla y si está Clasificado o Eliminado.
  3. Al final, las estadísticas: cuántos clasifican a la final y cuántos son del país "CL" (usa tu contarJugadoresQue de la etapa 5).

Requisitos: reporteTorneo NO debe repetir cálculos; debe invocar a las funciones de las etapas anteriores. Llámala una sola vez al final del archivo.

¿Necesitas una pista?
  • Es una arrow "orquestadora": adentro usa paraCadaJugador, puntajeTotal, promedio, medalla, clasificar y contarJugadoresQue.
  • Para la línea de cada jugador: paraCadaJugador((j) => console.log(\`${j.nombre}: ...\`)) combinando varias de tus funciones.
  • Para las estadísticas: contarJugadoresQue((j) => puntajeTotal(j) >= 400) y contarJugadoresQue((j) => j.pais === "CL").
  • No necesitas nada nuevo: TODO ya lo construiste. Esta etapa solo lo conecta.
