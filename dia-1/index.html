🎮 Laboratorio de Funciones

7 retos para entrenar TODO lo de funciones, y un Jefe Final que lo integra. Esta vez no se juega en una página: vas a resolver cada reto en tu propio archivo .js, lo vas a versionar con Git (un commit por reto), subirlo a GitHub y publicarlo en GitHub Pages. Funciones + Git + entrega real. Así se programa de verdad.

📜 Cómo se juega

  1. Primero arma tu proyecto (sección de abajo): carpeta, archivos .js, index.html y repositorio Git.
  2. Lee la misión de cada reto. Dice QUÉ lograr, no CÓMO. Eso lo descubres tú.
  3. Copia el "punto de partida" al archivo .js del reto y escribe ahí tu solución.
  4. Pruébalo en tu computador (con Node o con Live Server) y compara con la salida esperada.
  5. Cuando un reto funcione: haz un commit de ese archivo. Un reto = un commit con mensaje claro.
  6. Marca "Misión cumplida" y mira subir tu progreso.

🏅 Progreso: 0 / 8 misiones cumplidas

🚀 Arma y entrega tu proyecto

Esta actividad no se ejecuta aquí. Creas tu propio proyecto, lo versionas con Git, lo subes a GitHub y lo publicas en GitHub Pages. Sigue estos 6 pasos:

1 Crea tu carpeta con esta estructura

practica-funciones/
├── index.html        ← carga todos tus archivos .js
├── README.md         ← describe tu proyecto (buena práctica)
├── reto-1.js
├── reto-2.js
├── reto-3.js
├── reto-4.js
├── reto-5.js
├── reto-6.js
├── reto-7.js
└── jefe-final.js     ← el Jefe Final

2 Copia este index.html base

Pégalo tal cual en tu index.html. Carga todos tus .js; los resultados de los console.log se ven en la consola del navegador (tecla F12 → pestaña Console).

<!DOCTYPE html>
<html lang="es-CL">
<head>
  <meta charset="UTF-8" />
  <title>Práctica de Funciones</title>
</head>
<body>
  <h1>Práctica de Funciones — abre la consola (F12)</h1>

  <script src="reto-1.js"></script>
  <script src="reto-2.js"></script>
  <script src="reto-3.js"></script>
  <script src="reto-4.js"></script>
  <script src="reto-5.js"></script>
  <script src="reto-6.js"></script>
  <script src="reto-7.js"></script>
  <script src="jefe-final.js"></script>
</body>
</html>
Para probar en tu computador, abre index.html con la extensión Live Server de VS Code (clic derecho → Open with Live Server) y mira la consola (F12). Si tienes Node, también puedes probar cada archivo suelto con node reto-1.js en la terminal.

3 Inicializa Git en tu carpeta

Abre la terminal dentro de practica-funciones/ y crea el repositorio local:

git init
git add .
git commit -m "chore: estructura inicial del proyecto"
cada vez que termines un reto, haces un commit nuevo. Así tu historial cuenta la historia de tu trabajo, reto por reto. NO dejes todo para un solo commit gigante al final: eso es justo lo que NO queremos.

4 Resuelve cada reto y haz un commit por cada uno

Trabaja un reto, pruébalo, y cuando funcione versiónalo. El patrón que repites es siempre el mismo:

git add reto-1.js
git commit -m "feat: reto 1 - funcion presentar"

# ...resuelves el reto 2...
git add reto-2.js
git commit -m "feat: reto 2 - funcion multiplicar"

Revisa tu avance cuando quieras con git status (qué cambió) y git log --oneline (tu historial de commits).

5 Sube tu proyecto a GitHub

Opción A — Por la terminal (recomendada, refuerza Git)

Crea primero un repositorio vacío en GitHub llamado practica-funciones. Luego, dentro de tu carpeta:

git branch -M main
git remote add origin https://github.com/TU-USUARIO/practica-funciones.git
git push -u origin main

De ahí en adelante, cada vez que hagas commits nuevos, súbelos con un simple git push.

Opción B — Subiendo archivos por la web

Crea el repositorio en GitHub y arrastra TODOS los archivos (el index.html, los .js y el README.md). Funciona, pero pierdes la práctica de Git: usa esta opción solo si la terminal te falla.

6 Publica en GitHub Pages

  1. En tu repositorio de GitHub, entra a Settings.
  2. En el menú izquierdo, abre Pages.
  3. En Branch elige main y la carpeta / (root). Guarda.
  4. Espera ~1 minuto y entra al enlace que te entrega (algo como https://TU-USUARIO.github.io/practica-funciones/).
  5. Abre la consola (F12) en esa página: ahí se ven los resultados de tus funciones.
Lo que entregas es el enlace de GitHub Pages y el enlace del repositorio. El repo demuestra tu trabajo con Git (tus commits); la página demuestra que corre.
Nivel 1

Calentamiento: definir, invocar y retornar

Reto 1 · Tu primera función Nivel 1

Poder a entrenar: definir una función, recibir un parámetro y devolver con return.

  1. Crea una función llamada presentar que reciba un nombre.
  2. La función debe devolver (no imprimir) el texto: "Hola, me llamo ..." seguido del nombre.
  3. Invócala con dos nombres distintos y muestra ambos resultados con console.log.
¿Necesitas una pista?
  • La estructura es function nombre(parametro) { ... }.
  • Para devolver el texto usa return, y únelo con el nombre usando +.
  • Definir la función NO la ejecuta. Tienes que invocarla con paréntesis y pasarle el nombre.
reto-1.js — punto de partida
// Define aquí tu función presentar(nombre)


// Invócala con dos nombres distintos y muestra los resultados

Hola, me llamo Ana
Hola, me llamo Luis

Los nombres que uses son tu elección. Lo importante: que la función DEVUELVA el texto y tú lo muestres.

Al terminar: git add reto-1.js → git commit -m "feat: reto 1 - funcion presentar"

Reto 2 · La calculadora reutilizable Nivel 1

Poder a entrenar: parámetros, return y guardar el resultado en una variable.

  1. Crea una función multiplicar que reciba dos números y devuelva su producto.
  2. Guarda el resultado de multiplicar 6 y 7 en una variable llamada resultado y muéstrala.
  3. Muestra también el doble de ese resultado, pero sin volver a llamar la función (usa la variable que guardaste).
¿Necesitas una pista?
  • Dentro de la función: return a * b;.
  • Guardar el retorno: let resultado = multiplicar(6, 7);.
  • El doble es resultado * 2. Por eso guardar el valor te ahorra llamar la función de nuevo.

Parámetros con defecto y decisiones

Reto 3 · El despacho a domicilio Nivel 2

Poder a entrenar: parámetros con valor por defecto.

  1. Crea calcularEnvio(distancia, costoPorKm) que devuelva distancia * costoPorKm.
  2. El costoPorKm debe valer 500 por defecto.
  3. Pruébala dos veces: una para 10 km sin pasar el costo (debe usar 500), y otra para 10 km pasando 800.
¿Necesitas una pista?
  • El valor por defecto se escribe en la propia definición: function calcularEnvio(distancia, costoPorKm = 500).
  • El valor por defecto entra solo cuando NO pasas ese argumento.
  • Si pasas el segundo argumento (800), ese gana al valor por defecto.

Reto 4 · El semáforo de notas Nivel 2

Poder a entrenar: decidir dentro de una función y salir temprano con return.

  1. Crea clasificarNota(nota).
  2. Si la nota es 4.0 o más, debe devolver "Aprobado". Si es menor, "Reprobado".
  3. Usa return para salir de la función apenas sepas la respuesta.
  4. Pruébala con 5.5, 3.9 y 4.0 (ojo con este último: el 4.0 justo, ¿aprueba?).
¿Necesitas una pista?
  • Para "4.0 o más" usa >=, no solo >. Por eso el 4.0 justo SÍ aprueba.
  • Si entras al if y haces return "Aprobado", la función termina ahí: lo de abajo no corre.
  • No necesitas un else: si no aprobó, ya solo queda return "Reprobado".

Reto 5 · La caja fuerte (scope) Nivel 2

Poder a entrenar: entender que lo declarado dentro de una función no existe fuera.

  1. Crea una función secreto que declare dentro una variable local clave con el valor que quieras, y la devuelva.
  2. Invócala y muestra el resultado. Funciona ✅.
  3. Ahora, en una línea aparte, intenta mostrar clave directamente (fuera de la función) y ejecuta (en tu navegador o con Node).
  4. Observa el error en la consola. Escribe un comentario explicando por qué ocurrió.
¿Necesitas una pista?
  • Declara clave con let DENTRO de la función.
  • Al llamar console.log(clave) fuera, verás ReferenceError: clave is not defined.
  • Eso NO es un bug tuyo: es el scope protegiendo lo local. Fuera de la función, esa variable literalmente no existe.

Jefe Final

El integrador: lo juntas TODO

🏆 Jefe Final · Boleta de la cafetería Integrador

Aquí se combina TODO: definir varias funciones, parámetros, valor por defecto, return y funciones que se llaman entre sí. Funciones pequeñas, cada una con UNA tarea, trabajando en equipo. Eso es código limpio.

Vas a procesar la compra de una cafetería. Crea estas cuatro funciones:

  1. subtotal(precio, cantidad) → devuelve precio * cantidad. La cantidad por defecto es 1.
  2. aplicarIva(monto) → devuelve el monto con un 19% de IVA agregado.
  3. aplicarDescuento(monto, porcentaje) → resta ese porcentaje al monto. El porcentaje por defecto es 0.
  4. generarBoleta(producto, precio, cantidad, descuento) → la función orquestadora: usa las tres anteriores para calcular el total y devuelve un texto como "Café x2 = $5.950".

Requisitos obligatorios: usa parámetros, al menos un valor por defecto, return en todas, y que generarBoleta invoque a las otras tres (no repitas cálculos). Prueba con al menos dos compras distintas, una de ellas con descuento.

¿Necesitas una pista? (están en orden, abre solo la que te falte)
  • Subtotal: function subtotal(precio, cantidad = 1) { return precio * cantidad; }.
  • IVA: agregar 19% es monto + monto * 0.19 (o monto * 1.19).
  • Descuento: restar un porcentaje es monto - monto * (porcentaje / 100).
  • Orquestar: dentro de generarBoleta primero calcula el subtotal, pásalo a aplicarIva, ese resultado a aplicarDescuento, y arma el texto final con +.
  • Formato de pesos (opcional): "$" + valor.toLocaleString("es-CL") pone el separador de miles.
